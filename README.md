#### 项目声明

- 1. 关于版本兼容：建议您使用aseprite最新版本，不过即使是免费自编译版或是其他来源版本，我们都会做详尽的版本检查，对于不兼容的版本可能会导致部分插件无法使用（会提示您兼容性问题），后续可能会对多插件扩展做抽离或者其他操作会根据用户实际使用情况做处理。
- 2. 关于版权问题：该项目集合了各类开源项目资源，且对其做了一些适应性调整，代码均与源代码有偏差，如需原功能可通过扩展种的README查看信息及版权。
- 3. 项目打包：提供了windows下的简单打包，注意：该打包没有排除指定文件或目录功能。
- 4. 权限：扩展中的一些脚本会用到文件的存放读取等操作，aseprite也会相应的提示您是否授权，你当然可以拒绝，但相对应的该插件的功能将不可使用。
- 5. 扩展禁用：针对多类别官方并未提供具体的禁用api，所以这里采用的是比较粗暴的方式，直接读取扩展中代码并删除相关插件的引入，并生成禁用配置文件，当需要恢复时会读取该配置文件并恢复文件代码的引入，当然在此期间需要重启。
- 6. 软件翻译：作者基于其他开源翻译升级到了最新的实验版翻译，现有翻译中文简体、中文繁体，如有翻译错误可提供反馈，这边会尽快修复，感谢。

#### 项目目录

```plaintext
工作目录/
└─ FL-Extension/  // 扩展源文件目录
   ├─ pack_extension.lua  // 自动打包脚本
   ├─ package.json // 扩展配置目录
   ├─ lang/ // 扩展语言包资源目录
   ├─ config/ // 全局配置
   ├─ themes/ // 主题
   ├─ keys/ // 快捷键配置
   ├─ utils/ // 工具
   └─ scripts/ // 扩展程序目录
```

#### 开发声明

> 1. 外部开源协议引入的扩展，必须严格按照协议内容展示版权信息！
> 2. 脚本文件建议以文件夹形式存放，用于以后有可能出现的功能扩展。
> 3. 虽然并未对多语言严格要求，但建议是最好有多语言，且代码内部debug最好也要有。注释则是必须的，对于代码模块以及变量声明都应具体说明其作用和效应。

#### 扩展目录

- 字体：提供兼容中（繁）、日、韩、英字体。
- 多语言配置：语言更新至最新编译版（1.x.dev/v1.3.16-beta1），虽然官方已有多语言配置，但这里还是尽量对多语言的配置进行了翻译，也尽量与官方的翻译一致，这样也适用于一些免费编译的版本。
- 快捷键面板扩展，可以独立于系统快捷键不与系统快捷键存在冲突，使用指定快捷键或者选项中打开功能快捷键面板操作。
- 动画帧播放时绘制：可以在一组动画帧播放时使用画笔绘制。
- 自动生成动画帧：每一步绘制都自动生成动画帧。
- 像素块颜色统计：统计你文档中全部图层或单个图层的像素块颜色数量及比例。
- 自动水印生成：自动生成指定的文字水印或者图片水印甚至绘制好的精灵图水印。
- 自动描边：可以设置描边颜色与宽度，在选中的图层中自动为图层绘制指定配置的描边。
